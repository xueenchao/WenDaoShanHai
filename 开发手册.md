## 3. 应用入口与全局对象 (main.cpp)

`main.cpp` 中在文件作用域创建了全局对象：

```cpp
static Window gWindow("问道·山海", 1280, 720);
static Renderer gRenderer(gWindow);       // 绑定到 gWindow
static EventHandler gEventHandler;
static GameLoop gGameLoop(gEventHandler, 60);
static SceneManager gSceneManager(gEventHandler);
static Font* gFont = nullptr;             // 在 onInit 中 new，在 onCleanup 中 delete
static PlayerData gPlayerData;            // 玩家持久数据，跨场景共享
```

**GameLoop 四个回调**:

| 回调 | 时机 | 做什么 |
|------|------|--------|
| `onInit()` | 启动时调用一次 | 初始化 JsonLoader → DataManager → SDL → Window → Renderer → Font → 玩家数据 → 进入主菜单 |
| `onUpdate(dt)` | 每帧调用 | SceneManager 更新 → 检查退出/菜单转大世界 |
| `onRender()` | 每帧调用 | SceneManager 渲染 → present |
| `onCleanup()` | 退出时调用一次 | 清理 Font → TTF_Quit → Renderer → Window → SDL_Quit |

**启动顺序 (onInit)**:

1. `JsonLoader::initBasePath()` — 探测 exe 目录/父目录找 `data/`
2. `DataManager::getInstance().loadAll()` — 加载 8 个 JSON 文件
3. `SDL_Init` → `gWindow.create()` → `gRenderer.create()` → `gRenderer.setLogicalSize()`
4. `TTF_Init` → 加载字体（三级回退链）
5. 初始化玩家数据（技能、物品、本命法宝、符箓、门派）
6. `gSceneManager.pushScene(MainMenuScene)`

**场景转换**: `onUpdate` 中检查 `active->mName == "主菜单"` + `shouldStart()` 标志，调用 `switchScene(BigWorldScene)`。这是唯一通过名字判断场景类型的地方。

### 4.1 Window — 窗口管理

**头文件**: `Core/Window.h`

```cpp
// 创建窗口
Window window("标题", 1280, 720);
if (!window.create()) { /* 失败处理 */ }

// 基本查询
bool valid = window.isValid();
SDL_Window* raw = window.getRawWindow();
std::string title = window.getTitle();
int w, h;
window.getSize(w, h);  // 输出参数

// 修改属性
window.setTitle("新标题");
window.setSize(1920, 1080);
window.setFullscreen(true);
window.toggleFullscreen();

// 销毁（RAII，通常不需要手动调用）
window.destroy();
```

**要点**: Window 必须先 `create()` 才能使用。Renderer 依赖 Window 引用，所以 Renderer 构造前 Window 必须已创建。

### 4.2 Renderer — 渲染器

**头文件**: `Core/Renderer.h`

```cpp
// 构造（绑定到 Window）
Renderer renderer(window);
renderer.create();

// 逻辑分辨率（Letterbox 缩放）
renderer.setLogicalSize(1280, 720);  // 推荐：与 config.json viewport 一致
int lw, lh;
renderer.getLogicalSize(lw, lh);

// 绘制颜色
renderer.setDrawColor(255, 0, 0, 255);   // RGBA
renderer.clear();                          // 用当前颜色清屏
renderer.present();                        // 交换缓冲区

// 绘制图元（使用 float 坐标，对应 SDL_FRect）
renderer.drawLine(10.0f, 10.0f, 100.0f, 100.0f);
renderer.drawRect(50.0f, 50.0f, 200.0f, 100.0f);   // 空心矩形
renderer.fillRect(50.0f, 50.0f, 200.0f, 100.0f);   // 填充矩形
renderer.drawPoint(100.0f, 100.0f);

// 绘制纹理（三种重载）
SDL_Texture* tex = /* ... */;

// 1. 指定位置（原始大小）
renderer.drawTexture(tex, dstX, dstY);

// 2. 指定位置和大小（缩放）
renderer.drawTexture(tex, dstX, dstY, dstW, dstH);

// 3. 指定源矩形和目标矩形（裁剪+缩放）
renderer.drawTexture(tex,
    srcX, srcY, srcW, srcH,    // 源矩形（纹理上的区域）
    dstX, dstY, dstW, dstH);   // 目标矩形（屏幕上的区域）

// 旋转绘制
renderer.drawTextureRotated(tex, dstX, dstY, dstW, dstH,
    angle, centerX, centerY, SDL_FLIP_NONE);

// 渲染目标（用于离屏渲染）
renderer.setRenderTarget(renderTexture);
renderer.setRenderTarget(nullptr);  // 恢复默认

// 视口裁剪
renderer.setViewport(x, y, w, h);
renderer.resetViewport();
```

**要点**:

- 坐标全部使用 `float`（SDL3 的 `SDL_FRect`）
- 逻辑分辨率配合 `setLogicalSize()` 使用，SDL3 自动处理 Letterbox 缩放
- 每帧流程: `clear()` → 绘制 → `present()`

### 4.3 Texture — 纹理/图片

**头文件**: `Core/Texture.h`

```cpp
Texture tex;

// 从文件加载
tex.loadFromFile(renderer, "assets/player.png");

// 从内存加载
tex.loadFromMemory(renderer, dataPtr, dataSize);

// 创建空白纹理（用于离屏渲染）
tex.createBlank(renderer, 256, 256, SDL_TEXTUREACCESS_TARGET);

// 从 Surface 创建
tex.createFromSurface(renderer, surface);

// 查询
bool valid = tex.isValid();
int w = tex.getWidth();
int h = tex.getHeight();
SDL_Texture* raw = tex.getRawTexture();

// 颜色调制（着色/半透明）
tex.setColorMod(255, 0, 0);    // 红色着色
tex.setAlphaMod(128);           // 半透明
tex.setBlendMode(SDL_BLENDMODE_BLEND);

// 绘制（通过 Renderer）
renderer.drawTexture(tex.getRawTexture(), x, y);

// 销毁
tex.destroy();
```

**要点**:

- 支持移动语义（`std::move`），但禁止拷贝
- `loadFromFile` 支持 PNG/JPG/BMP/WEBP（通过 SDL3_image）
- `renderText` 返回的 `SDL_Texture*` 需要手动 `SDL_DestroyTexture()`

### 4.4 Font — 字体渲染

**头文件**: `Core/Font.h`

```cpp
Font font;

// 加载字体
font.load("C:/Windows/Fonts/msyh.ttc", 16.0f);  // SDL3 用 float 字号

// 查询
bool valid = font.isValid();
float size = font.getFontSize();

// 修改字号（重新打开同一文件，不同大小）
font.setFontSize(20.0f);

// 样式
font.setStyle(TTF_STYLE_BOLD);
TTF_FontStyleFlags style = font.getStyle();

// 渲染文字为纹理（返回 SDL_Texture*，调用者必须 SDL_DestroyTexture）
SDL_Texture* textTex = font.renderText(renderer, "你好世界", 255, 255, 255);
// ... 使用 textTex 绘制 ...
SDL_DestroyTexture(textTex);

// 三种渲染质量
// RenderQuality::Solid    — 最快，无抗锯齿
// RenderQuality::Shaded   — 有背景色，抗锯齿
// RenderQuality::Blended  — 最慢，全抗锯齿，透明背景（默认推荐）

// 带背景色的渲染
SDL_Texture* shaded = font.renderTextShaded(renderer, "文本",
    255, 255, 255,   // 前景色
    0, 0, 0);         // 背景色

// 自动换行渲染
SDL_Texture* wrapped = font.renderTextWrapped(renderer, "很长的文本...",
    255, 255, 255, 300, Font::RenderQuality::Blended);  // wrapWidth=300

// 测量文字尺寸（用于布局计算）
int textW, textH;
font.measureText("你好", textW, textH);
```

**要点**:

- SDL3 的 `TTF_OpenFont()` 用 `float ptsize`，不是 SDL2 的 `int`
- `renderText` 每次调用都创建新 `SDL_Texture*`，用完必须 `SDL_DestroyTexture`
- UILabel 内部每帧都会重新渲染文字纹理，这是当前的性能瓶颈之一
- 字体加载有三级回退链：config.fontPrimary → config.fontFallback → msyh.ttc → simsun.ttc

### 4.5 Audio — 音频系统

**头文件**: `Core/Audio.h`

```cpp
Audio audio(44100, 2);  // 频率, 声道数
audio.init();

// 加载音频
audio.loadAudio("bgm", "assets/bgm_main.ogg", true);   // predecode=true 适用于短音效
audio.loadAudio("sfx_hit", "assets/hit.wav", true);

// 播放
audio.play("sfx_hit");                   // 播放一次
audio.play("bgm", -1);                   // 循环播放（-1=无限循环）

// 按标签播放（可分组控制）
audio.playWithTag("bgm", "music", -1);   // 标签 "music"
audio.playWithTag("footstep", "sfx");     // 标签 "sfx"

// 停止
audio.stopAll();                          // 停止所有
audio.stopAll(500);                       // 500ms 淡出
audio.stopByTag("music");                 // 停止标签组
audio.stopByTag("music", 500);            // 淡出停止

// 暂停/恢复
audio.pauseAll();
audio.pauseByTag("music");
audio.resumeAll();
audio.resumeByTag("music");

// 音量（0.0~1.0）
audio.setMasterVolume(0.8f);
audio.setTagVolume("music", 0.5f);
float vol = audio.getMasterVolume();

// 卸载
audio.unloadAudio("sfx_hit");
audio.unloadAllAudio();

// 关闭
audio.shutdown();
```

**要点**:

- 使用 SDL3_mixer 的新 API（`MIX_Mixer*` / `MIX_Audio*` / `MIX_Track*`），不是旧版 `Mix_Chunk*` / `Mix_Music*`
- `predecode=true` 适合短音效（一次性解码到内存），`false` 适合长 BGM（流式播放）
- 标签（tag）系统可将音频分组，便于批量控制（如"静音背景音乐但保留音效"）

### 4.6 EventHandler — 输入事件

**头文件**: `Core/EventHandler.h`

```cpp
EventHandler eh;

// 事件轮询（在 GameLoop 中自动调用）
bool running = eh.pollEvents();  // 返回 false 表示收到退出信号

// 键盘状态（双帧比较，检测"刚按下/刚松开"）
bool held = eh.isKeyPressed(SDL_SCANCODE_W);            // 当前帧是否按住
bool justDown = eh.isKeyJustPressed(SDL_SCANCODE_W);    // 刚按下（本帧按下，上一帧未按下）
bool justUp = eh.isKeyJustReleased(SDL_SCANCODE_W);     // 刚松开

// 鼠标状态
float mx, my;
eh.getMousePosition(mx, my);                            // 鼠标位置（逻辑坐标）
bool leftHeld = eh.isMouseButtonPressed(SDL_BUTTON_LEFT);
bool leftJust = eh.isMouseButtonJustPressed(SDL_BUTTON_LEFT);
bool leftJustUp = eh.isMouseButtonJustReleased(SDL_BUTTON_LEFT);

// 鼠标滚轮
float wx, wy;
eh.getMouseWheel(wx, wy);

// 窗口事件
bool quit = eh.isQuitRequested();
bool resized = eh.isWindowResized();

// ⚠️ 必须在每帧事件处理完毕后调用！
eh.update();  // 将 current 状态复制到 previous，为下一帧的 JustPressed/JustReleased 做准备
```

**要点**:

- `isKeyJustPressed` / `isKeyJustReleased` 依赖双帧状态比较，必须每帧末尾调用 `update()`
- 鼠标坐标已经是逻辑坐标（经过 `setLogicalSize` 变换后的）
- `pollEvents()` 返回 `false` 表示收到 `SDL_EVENT_QUIT`

### 4.7 GameLoop — 主循环

**头文件**: `Core/GameLoop.h`

```cpp
GameLoop loop(eventHandler, 60);  // 60 FPS

// 设置回调
loop.setOnInit([]() -> bool { /* 初始化 */ return true; });
loop.setOnUpdate([](float dt) { /* 更新逻辑 */ });
loop.setOnRender([]() { /* 渲染 */ });
loop.setOnCleanup([]() { /* 清理 */ });

// 启动（阻塞，直到 stop() 被调用）
loop.run();

// 从回调中停止
loop.stop();

// 查询
bool running = loop.isRunning();
int fps = loop.getTargetFPS();
float currentFPS = loop.getCurrentFPS();
float dt = loop.getLastDeltaTime();

// 动态修改 FPS
loop.setTargetFPS(120);
```

**要点**:

- `deltaTime` 上限 0.1s，防止长时间暂停后的巨大 dt
- 四个回调中只有 `onInit` 返回 `bool`（失败可终止启动），其余返回 `void`
- 实际执行顺序: `onInit` → 循环{ `pollEvents` → `onUpdate(dt)` → `onRender()` → 计算FPS/限帧 } → `onCleanup`

### 4.8 Camera2D — 2D 相机

**头文件**: `Core/Camera2D.h`

```cpp
Camera2D camera(1280.0f, 720.0f);  // 视口宽高

// 设置位置（世界坐标）
camera.setPosition(500.0f, 300.0f);
camera.setPosition(glm::vec2(500.0f, 300.0f));
glm::vec2 pos = camera.getPosition();

// 相对移动
camera.move(10.0f, 0.0f);  // 向右移动10像素

// 缩放
camera.setZoom(2.0f);
camera.zoomIn(0.1f);       // 放大0.1
float z = camera.getZoom();  // 范围 0.1~10

// 旋转
camera.setRotation(45.0f);
float rot = camera.getRotation();

// 边界约束
camera.setBounds(0, 0, 2560, 1440);  // 限制相机不超出世界范围
camera.clearBounds();

// 平滑跟随（推荐）
glm::vec2 playerPos = /* ... */;
camera.follow(&playerPos, 5.0f);   // 跟随指针指向的位置，平滑速度 5.0
camera.snapToTarget();               // 立即跳到目标位置（无平滑）
camera.stopFollow();
bool following = camera.isFollowing();

// ⚠️ 每帧调用 update
camera.update(deltaTime);

// 坐标转换
glm::vec2 screenPos = camera.worldToScreen(worldX, worldY);
glm::vec2 worldPos = camera.screenToWorld(screenX, screenY);

// 视口范围
AABB viewBounds = camera.getViewBounds();
glm::vec2 vpSize = camera.getViewportSize();
```

**要点**:

- `follow()` 接受 `glm::vec2*` 指针，每帧 `update()` 时读取指针位置做指数衰减插值
- `worldToScreen` / `screenToWorld` 用于将鼠标点击转为世界坐标（或将物体位置转为屏幕坐标）
- 缩放范围 0.1~10，边界约束防止相机显示世界外区域

### 4.9 CollisionSystem — 碰撞检测

**头文件**: `Core/CollisionSystem.h`

```cpp
CollisionSystem cs(64.0f);  // 空间哈希网格单元大小

// 预定义碰撞层（位掩码）
// CollisionLayer::None=0, Player=1, Enemy=2, Terrain=4,
// Projectile=8, Item=16, Trigger=32, All=0xFFFFFFFF

// 添加碰撞体
AABB box = {100, 100, 32, 32};  // x, y, w, h
uint32_t id = cs.addCollider(box, CollisionLayer::Player, CollisionLayer::Enemy);

// 更新碰撞体位置
cs.updateCollider(id, {150, 100, 32, 32});

// 启用/禁用
cs.setActive(id, false);

// 点查询
std::vector<uint32_t> hits = cs.queryPoint(mouseX, mouseY, CollisionLayer::Enemy);

// 区域查询
AABB area = {50, 50, 200, 200};
std::vector<uint32_t> inArea = cs.queryAABB(area, CollisionLayer::Item);

// 移动检测（检测新位置是否碰撞）
std::vector<uint32_t> blockers = cs.checkMove(id, newBounds);

// 删除碰撞体
cs.removeCollider(id);

// 重建空间哈希（大范围修改后调用）
cs.rebuild();

// 清空
cs.clear();

// 查询碰撞体信息
const Collider* c = cs.getCollider(id);
size_t count = cs.getColliderCount();
```

**AABB 工具方法**:

```cpp
AABB a = {0, 0, 100, 100};
AABB b = {50, 50, 100, 100};
bool overlap = a.overlaps(b);       // AABB 重叠检测
bool inside = a.contains(30, 30);   // 点包含检测
float cx = a.centerX();             // 中心点
float cy = a.centerY();
```

**要点**:

- `layer` 是自身所属层，`mask` 是关心哪些层（位运算）
- 空间哈希网格加速查询，大范围位置更新后可调用 `rebuild()`
- 当前项目中战斗系统直接用 GridMap 管理，CollisionSystem 主要用于大世界

### 4.10 Scene / SceneManager — 场景系统

**头文件**: `Core/Scene.h`

#### Scene 基类

```cpp
class MyScene : public Scene {
public:
    MyScene() : Scene("我的场景") {}

    void onEnter() override { /* 进入场景时初始化 */ }
    void onExit() override { /* 离开场景时清理 */ }

    void onUpdate(float deltaTime) override {
        // 每帧更新逻辑（纯虚函数，必须实现）
    }

    void onRender(Renderer& renderer) override {
        // 每帧渲染（纯虚函数，必须实现）
    }

    void onEvent(const SDL_Event& event) override {
        // 可选：处理 SDL 原始事件
    }
};
```

#### SceneManager

```cpp
SceneManager mgr(eventHandler);

// 场景栈操作
mgr.pushScene(std::make_unique<MyScene>());       // 压入（暂停当前场景）
mgr.switchScene(std::make_unique<MyScene>());     // 替换（销毁当前场景）
mgr.popScene();                                    // 弹出当前场景（恢复上一个）

// ⚠️ 从场景内部弹出自身（安全方式）
mgr.requestPopScene();  // 延迟到 onUpdate 结束后执行，避免 use-after-free

// 查询
Scene* active = mgr.getActiveScene();
bool empty = mgr.isEmpty();
bool quit = mgr.isQuitRequested();

// 每帧调用
mgr.onUpdate(deltaTime);
mgr.onRender(renderer);
mgr.onCleanup();
```

**场景栈示例**:

```
MainMenuScene                    ← switchScene 替换
  └→ BigWorldScene               ← pushScene 压入
       └→ CombatScene            ← pushScene 压入
            └→ (胜利) requestPopScene → 回到 BigWorldScene
       └→ SectScene              ← pushScene 压入（按 T 键）
            └→ (ESC) requestPopScene → 回到 BigWorldScene
```

**要点**:

- **永远不要在 `onUpdate` 内部调用 `popScene()`**，必须用 `requestPopScene()`
- `pushScene` 暂停当前场景（不调用 `onExit`），`popScene` 恢复时调用 `onEnter`
- `switchScene` 先 `onExit` 当前场景，再销毁，再压入新场景

- 控件坐标是相对于父元素的，`getWorldPosition()` 从根累计偏移得到屏幕绝对位置

### 7.5 已知陷阱与注意事项

#### UIManager 原始指针崩溃

**问题**: `UIManager` 存有 `mHoveredElement` 和 `mPressedElement` 原始指针。如果在鼠标按下时销毁按钮元素，下一次 `handleMouseUp` 会访问悬空指针。

**解决**: 销毁任何 UI 元素之前，必须调用 `mUIManager.resetInteractionState()`。

```cpp
// 正确
mUIManager.resetInteractionState();
mUIManager.clear();
// 或
mUIManager.resetInteractionState();
mUIManager.getRoot()->mChildren.clear();
```

#### 场景自我弹出

**问题**: 在场景的 `onUpdate` 中直接调用 `popScene()` 会导致正在执行的方法访问已销毁的对象。

**解决**: 使用 `requestPopScene()`，它会延迟到 `onUpdate` 返回后再执行。

```cpp
// 错误！
mSceneManager->popScene();  // 自杀式调用

// 正确
mSceneManager->requestPopScene();  // 安全延迟弹出
```

#### 覆盖层可见性

**问题**: 切换 `mInventoryOpen` 布尔标志后，如果不同步 UI 元素的 `mVisible`，覆盖层不会显示/隐藏。

**解决**: 每次切换标志后，调用对应的 `update*Display()` 方法同步 `mVisible`。

```cpp
mInventoryOpen = !mInventoryOpen;
updateInventoryDisplay();  // 必须调用！
```

#### PlayerData 内存泄漏

**问题**: `equipItem()` 在装备栏已占有时直接覆盖指针，旧 Item 未被 delete。

**解决**: 先 `unequipSlot()` 再 `equipItem()`，或者修复 `equipItem` 使其先卸下旧装备。

```cpp
// 安全做法
playerData.unequipSlot(EquipSlot::Weapon);  // 先卸下
playerData.equipItem(index);                 // 再装备
```

#### Skill 按钮重建

**问题**: CombatScene 中技能按钮如果每帧销毁重建，UIManager 的悬停/按下指针会悬空。

**解决**: 用变化追踪（`mLastActiveUnitId`, `mLastPhase`, `mLastSkillCount`）判断是否需要重建。只重建结构变化时才销毁重建；纯视觉更新（如高亮色）直接修改现有元素属性。

#### 字体加载回退链

**问题**: 如果配置的字体不存在，游戏应仍能运行。

**解决**: 已有三级回退链在 `onInit` 中实现：`fontPrimary` → `fontFallback` → `msyh.ttc` → `simsun.ttc`。

  1. PlayerData::equipItem() 中的内存泄漏

  文件：src/Game/Core/PlayerData.h:116-120
