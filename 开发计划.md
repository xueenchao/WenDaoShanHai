## 一、当前优先：完成第二阶段「初识斗法」（剩余 ~25%）

### 1.1 技能区域效果（Cross / Square）
- [ ] `CombatScene::executeAttack()` 实现十字形 (Cross) 范围技能
- [ ] 实现方形 (Square) 范围技能
- [ ] 技能面板显示范围预览（高亮受影响格子）

### 1.2 战斗失败与死亡处理
- [ ] 玩家单位 HP 归零 → 显示"战斗失败"界面
- [ ] 失败后选项：返回大世界（HP/SP 恢复至一定比例）
- [ ] 单位死亡动画 / 状态标记

### 1.3 升级系统
- [ ] `PlayerData::xp` 满后触发 `levelUp()`
- [ ] 升级时提升属性（HP、SP、攻击、防御、速度）
- [ ] 升级提示 UI（战斗中 / 大世界）

### 1.4 符箓系统完善
- [ ] 实现召唤类符箓（Summon 类型）——生成临时傀儡单位
- [ ] 敌人 AI 可使用符箓
- [ ] 符箓效果从 JSON 数据驱动（当前硬编码）

### 1.5 战斗 UI 打磨
- [ ] 大世界角色面板显示 HP/SP 条
- [ ] 战斗胜利界面（经验获取、掉落物品展示）
- [ ] 战斗场景入场/退场过渡动画

---

## 二、第三阶段「机缘首获」——词条与装备系统

这是下一个大阶段，核心是建立 Diablo-like 随机装备驱动循环。

### 2.1 词条系统（Affix）
- [ ] 设计词条数据结构（`AffixDef`）：名称、触发时机、效果类型、数值范围、稀有度
- [ ] 词条 JSON 配置文件 `data/affixes.json`
- [ ] `AffixEffect` 基类 + 工厂模式
- [ ] 事件管理器：`ON_ATTACK`、`ON_KILL`、`ON_HIT`、`ON_EQUIP` 等触发时机
- [ ] 装备掉落时随机生成 1-4 条词条
- [ ] 装备 UI 显示词条列表及稀有度颜色

### 2.2 战利品系统
- [ ] `data/loot_tables.json` ——按敌人/区域配置的掉落表
- [ ] 掉落表支持：物品池、权重、数量范围、稀有度权重
- [ ] 战斗胜利时从掉落表抽取而非硬编码

### 2.3 装备系统深化
- [ ] 神通玉简（技能圣物）——装备后赋予额外技能
- [ ] 词条预览与对比（装备前 vs 装备后属性变化）
- [ ] 装备稀有度分级（白/蓝/金/暗金）

### 2.4 BD（Build）构筑验证
- [ ] 至少 2 个可玩的 BD 方向：火系爆发流、金系破甲流
- [ ] 词条联动效果验证：火球术 + 燃烧扩散、金属性 + 破甲叠加
- [ ] BD 切换成本（更换神通玉简 = 更换技能组）

## 三、第四阶段「山海初现」——御兽系统

### 3.1 神兽捕获
- [ ] `BeastDef` 数据结构（名称、属性、技能、资质、血脉）
- [ ] `data/beasts.json` 配置文件
- [ ] 战斗中使用灵兽环捕获逻辑（血量越低成功率越高）
- [ ] 捕获动画与提示

### 3.2 神兽养成
- [ ] 神兽等级、经验、资质系统
- [ ] 血脉觉醒：吞噬材料提升能力
- [ ] 神兽技能学习 / 进化
- [ ] 神兽管理 UI（查看、喂养、释放）

### 3.3 神兽战斗 AI
- [ ] `PetAI` 有限状态机（Idle / Follow / Attack / UseSkill / Flee）
- [ ] 指令模式：集火、保护、待命、释放技能
- [ ] 神兽在网格上的独立行动单位
- [ ] 神兽属性与五行克制联动

### 3.4 宠物关联词条
- [ ] 宠物继承角色暴击伤害比例
- [ ] 宠物击杀触发角色增益
- [ ] JSON 驱动宠物关联效果

## 四、第五阶段「洞府探秘」——深度系统

### 4.1 多层副本
- [ ] 副本地图生成（随机房间 + 走廊）
- [ ] 副本层级递进（难度、敌人等级递增）

### 4.3 阵法系统
- [ ] `FormationDef` 数据结构
- [ ] 战前布阵：在阵眼插旗，区域内效果生效

- [ ] 阵图学习与阵旗消耗

## 五、技术债务（穿插进行）

| 项目 | 优先级 | 说明 |
|------|--------|------|
| PlayerData 智能指针迁移 | 高 | `Item*` / `LifeboundTreasure*` 裸指针 → `unique_ptr` |
| 随机数种子初始化 | 高 | `srand()` 在启动时调用一次 |
| EquipItem 内存泄漏修复 | 中 | 重新装备时 delete 旧 Item |
| 测试框架搭建 | 中 | GoogleTest + 核心战斗逻辑单元测试 |
| CI/CD | 低 | GitHub Actions 自动构建 |
